VANTAGENS DO HTML

1. Torna a internet mais rápida: desenvolvedores vem usando HTML5 para reduzir o tamanho de arquivos e tornar a experiência do usuário mais limpa.
2. HTML5 tem Canvas que é uma poderosa ferramenta para a criação de conteúdo gráfico. Com Canvas é mais fácil criar animações, desenhos e outros elementos visuais complexos sem usar aplicativos externos.
3. Cada vez mais navegadores dão suporte ao HTML5. Quer saber se o seu navegador é compativel com HTML5? Vá para html5test.com e verifique, caso não seja, é melhor atualizar a versão de seu navegador.
4. Um dos grandes diferenciais do HTML5 é sua capacidade de vídeo. Desenvolvedores podem integrar vídeos e HTML5 sem a necessidade de plugins. E como temos cada vez mais navegadores que são compatíveis ao HTML5, vemos um rápido crescimento na porcentagem de vídeos online.
5. O mesmo acontece com sons. HTML5 vem com uma ferramenta de suporte poderosa para elementos de audio.
6. HTML5 está disponível em diversas plataformas e sua performance é ótima em PC's, dispositivos móveis e tablets.

DESVANTAGENS DO HTML

1. O modo WYSIWYG nem sempre é um retracto real da página, principalmente com o uso de alinhamentos em tabelas, células e CSS.
2. O modo de visualização (preview) incluído não corresponde a nenhum browser exactamente (nem permite a sua inclusão - como é o caso do Homesite) devendo pois ser testado em cada um deles.
3. O modo de procura e substituição de termos é somente página por página obrigando à abertura individual dos ficheiros, não possibilitando assim uma troca rápida de expressões. A versão seguinte (XP), já disponível em Versão Beta já permite a troca em ficheiros múltiplos mas somente de uma linha, não chegando ao poder da substituição fornecido pelo Homesite.
4. O uso de instruções do tipo
e posicionamento pelo uso de CSS não é apoiado pelo modo WYSIWYG.
5. O modo de edição em linha (HTML) não tem ajuda nos comandos e este modo é necessário para muitas correcções que não são possíveis no modo de edição WYSIWYG (normal)
6. Não fornece nenhuma ferramenta de apoio para a construção de CSS
7. produz muitos ficheiros que podem não ser usados (por exemplo pelo uso de temas) o que pode fazer ultrapassar a quota disponível.
8. o modo de publicação possibilita a publicação das somente das alterações, mas estas são verificadas pela comparação dos ficheiros o que torna esta opção morosa. Na versão XP será também possível o uso do TimeStamp para detectar as alterações.
9. Quando se faz "cut-paste" o Frontpage tenta usar as fontes de origem o que pode não ser o que queremos. a versão XP usando o novo sistema de "cut-paste" do Office XP permite a copia do texto sem formatações.
10. Não permite a gravação de todos os ficheiros abertos (Save All). A versão seguinte já possuirá essa opção.
11. Não verifica se nome dos ficheiros é compatível com o sistema Unix
12. O FrontPage deixa a sua assinatura nas páginas que edita (2 linhas em ) o que pode ser considerado uma tentativa de violação de privacidade
13. O produto implica o uso do Office.
VANTAGENS DE USAR CSS
  • Com CSS economiza-se tempo de criação e manutenção do site (o código de formatação fica isolado em um único arquivo CSS).
  • Redução do código HTML.
  • Os Navegadores carregam mais rápido a página, podendo chegar até a 70%.
  • Maior eficiência no gerenciamento do Layout.
  • Com recursos mais avaçados de CSS pode-se conseguir um design sofisticado sem utilização de imagens e tabelas.

    Misturar de novo o conteúdo e a apresentação:

    Quando se desenha uma web com CSS pode-se mudar seu aspecto radicalmente apenas com trocar a folha de estilos, inclusive a maneira como os elementos se posicionam na página. No entanto, quando se utiliza um framework estamos colocando classes (de CSS) que indicam onde vão ser posicionados os elementos em uma grade e, se quisermos mudar a posição desses elementos mais adiante nos veremos obrigados a trocar o código HTML da página e colocar outras classes CSS, que permitam situá-los em outros pontos dessa grade.

    Ter código CSS que nunca é utilizado:

    O framework CSS contém diversos códigos CSS úteis em casos gerais, porém o certo é que quando você desenha uma página web uma boa parte desse código não vai ser nunca usado . Ou seja, ao utilizar um framework estamos carregando desnecessariamente o peso em bytes de nosso código CSS. Isto pode ser consertado "à mão" limpando o CSS do framework, eliminando código que não cheguemos a utilizar em nenhuma página do site.

    Curva de aprendizagem mais lenta:

    Embora aprender a usar um framework CSS não seja nada complicado, você terá que conhecer suas particularidades para poder tirar proveito dele . Terá também que aprender a desenhar de uma maneira determinada, para usar a grade com a que se posicionam os elementos. Tudo isto, insisto, não é difícil, mas leva seu tempo. Tanto é assim que, se você for desenhar apenas uma web, talvez demore mais tempo em aprender a trabalhar com o framework CSS do que demorará em desenhá-la sem o utilizar. Resumindo, você só tirará proveito do framework quando conhecê-lo, ou seja no segundo, ou nos desenhos seguintes que você realize com ele.
